Commands & TLO

Last Updated: 2026-04-20

Slash Commands

All commands are registered under /mydps and dispatched via a single command handler function MyDPSCommand(). The handler reads the first argument with GetArg() and matches it case-insensitively using ci_equals(). If no argument is given, the main window is toggled.

The help text printed for any unrecognized argument lists all available subcommands:

[MQMyDPS] Commands: start, stop, reset, report, show, hide, spam, lock, config
CommandActionNotes
/mydps Toggle main report window No argument: flips showMainWindow
/mydps show Show main report window Sets showMainWindow = true
/mydps hide Hide main report window Sets showMainWindow = false
/mydps config Toggle config window Flips showConfigWindow
/mydps spam Toggle combat spam overlay Flips showCombatSpam, prints confirmation to chat
/mydps lock Toggle spam window click-through Flips settings.spamClickThrough. When on (default), the spam overlay passes all mouse input to the game. When off, the overlay is draggable. Prints "locked" or "unlocked" to chat.
/mydps start Start damage tracking Sets tracking = true. If no session damage has been recorded yet, also resets sessionStartTime to now so the DPS calculation starts from this moment.
/mydps stop Stop damage tracking Sets tracking = false. Existing data is preserved. OnPulse and OnIncomingChat processing also stop immediately (both check the flag before doing any work).
/mydps reset Clear all data Calls ResetAll(): clears recentRecords, currentTargets, currentHealTargets, battleHistory, activeDoTs, all counters, and resets sessionStartTime.
/mydps report Print session summary to chat (and MQMyChat if loaded) Prints session DPS, formatted total damage, and hit count to the EverQuest chat window in MQ color codes. If MQMyChat is loaded, the same formatted message is also sent to the "MyDPS" channel. Format: [MQMyDPS] Session DPS: X.X | Total: Xk | Hits: X

The lock Command in Detail

When the spam window is locked (spamClickThrough = true), ImGui creates it with the ImGuiWindowFlags_NoInputs flag. This tells the OS-level window message handler that this region should be treated as "transparent to input" — mouse clicks fall through to the EverQuest window underneath. This is the correct default behavior during combat because you do not want to accidentally drag a UI element when trying to click on a mob.

When you want to reposition the spam window, use /mydps lock to temporarily disable click-through, drag the window to the desired position, then use /mydps lock again to re-enable it. The setting is saved to the INI file when settings are saved.

Top Level Object: ${MyDPS}

What Is a TLO?

A Top Level Object (TLO) is the mechanism by which a MacroQuest plugin exposes data to MQ2 macros and Lua scripts. When you write ${MyDPS.DPS} in a macro, MQ2 finds the MyDPS TLO by name, calls its GetMember("DPS") function, and substitutes the result into the macro line. The TLO system provides a uniform interface so all plugins expose data the same way.

MQMyDPS registers its TLO in InitializePlugin() via AddTopLevelObject("MyDPS", MyDPSType::dataMyDPS) and removes it in ShutdownPlugin(). If the plugin is not loaded, ${MyDPS} returns nothing (macro authors should check ${MyDPS.Version} to confirm the plugin is active).

Target Type: ${MyDPS.Target[id]}

How Target Lookup Works

When you write ${MyDPS.Target[1234]}, MQ2 calls MyDPSType::GetMember("Target", "1234"). The handler parses the index as an integer with strtol(). If parsing succeeds, it calls FindTargetBySpawnID(1234) which does a direct hash map lookup in currentTargets. If parsing fails (the index is not a number), it calls FindTargetByName(index) which does a linear search by name.

If found, the spawn ID is stored in Dest.Int and the type is set to pMyDPSTargetType. Subsequent member accesses (e.g., .TotalDamage) call MyDPSTargetType::GetMember(), which retrieves the spawn ID from VarPtr.Int and looks up the live TargetDamageData again. This means the target data is always fresh — no caching at the TLO level.

Target lookup only searches currentTargets. The TLO cannot access historical battle data. If you want to query a mob from a previous battle, that data is in battleHistory, which is not exposed via TLO at this time.

Members of ${MyDPS.Target[id]}

MemberReturn typeDescription
Namestring The mob's display name as it appeared in the chat line when first hit.
SpawnIDint The spawn ID used as the map key. Positive values are real game spawn IDs. Negative values are synthetic fallbacks assigned when no real ID was resolvable.

ToString

Returns the target's display name. This means /echo ${MyDPS.Target[1234]} prints the mob's name, and /echo ${MyDPS.Target[1234].Name} gives the same result.

Usage Examples

Macro Syntax

/echo My DPS: ${MyDPS.DPS}
/echo Total Damage: ${MyDPS.TotalDamage}
/echo In Combat: ${MyDPS.InCombat}
/echo Target Count: ${MyDPS.TargetCount}

/echo Target by spawn ID:   ${MyDPS.Target[1234].Name} - ${MyDPS.Target[1234].TotalDamage}
/echo Target by name:       ${MyDPS.Target[a skeleton].DPS}

/echo Battle Count: ${MyDPS.BattleCount}
/echo Battle DPS:   ${MyDPS.BattleDPS}

${MyDPS.Reset}

Lua Usage

local dps       = mq.TLO.MyDPS.DPS()
local inCombat  = mq.TLO.MyDPS.InCombat()
local targetDmg = mq.TLO.MyDPS.Target(1234).TotalDamage()
local targetName = mq.TLO.MyDPS.Target("a skeleton").Name()

if mq.TLO.MyDPS.InCombat() then
    print("Battle DPS: " .. mq.TLO.MyDPS.BattleDPS())
end

Checking if the Plugin is Loaded

/if (${MyDPS.Version} == 0) {
    /echo MQMyDPS is not loaded.
    /return
}
