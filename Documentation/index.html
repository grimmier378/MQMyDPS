What is MQMyDPS?

MQMyDPS is a native C++ MacroQuest plugin that tracks your personal DPS (damage per second) and healing output in EverQuest. It is a port and evolution of MyUI's mydps.lua module, rewritten in C++ for better performance and deeper integration with MacroQuest's game state APIs.

The fundamental challenge in building a DPS tracker for EverQuest is that the game's combat chat text never includes spawn IDs. If you fight two mobs both named "a skeleton", every hit message just says "a skeleton" — there is no way to distinguish them from chat text alone. The Lua version of this tracker had this limitation. The C++ version overcomes it by reading live game state directly: your current target's spawn ID, your pet's attack target, the spell you just finished casting, and your extended target (XTarget) list.

Here is what the plugin provides:

  • Real-time combat spam overlay: A floating window showing recent damage and heal events, fully color-coded by event type, with configurable font scaling. Newest events appear at the top. Can be set click-through so it never interferes with gameplay.
  • Battle tracking: Automatic combat state detection using GetCombatState(). Battles are finalized after a configurable delay when combat ends, preserving per-spawn breakdowns in history.
  • Per-spawn damage attribution: Each damage event is resolved to a specific spawn ID using a multi-strategy approach — direct target check for melee, pet's target for pet damage, cast tracking registry for DoTs, XTarget scan for damage shields, and name-based fallback.
  • Pet and non-melee breakdown: Pet damage (melee and spells) and your own non-melee (direct damage spells) are tracked as separate sub-totals in both the current battle view and the history table.
  • Healing tracking: Direct heals and critical heals are recorded per player target. The Healing tab shows aggregated session heals per player, sorted by total, with a pie chart breakdown.
  • Battle history: Completed battles are stored with per-target and per-heal-target breakdowns. The History table has 12 columns (hideable, reorderable), with expandable rows per spawn and heal target sections.
  • Full color-coded UI: Stats values in the Current Battle tab are colored by data type. Target names use distinct colors: damage targets in reddish pink, heal targets in light blue. The menu bar DPS counter is also color-coded.
  • Graphs: ImPlot-powered DPS line graphs and damage bar charts across battles, plus pie charts showing damage distribution by target.
  • MQMyChat integration: If MQMyChat is loaded, battle summaries, heal reports, and the /mydps report command output are automatically sent to a dedicated "MyDPS" channel in MQMyChat using MQ color codes.
  • TLO API: Exposes DPS data to macros and Lua scripts via ${MyDPS.DPS}, ${MyDPS.Target[id].TotalDamage}, etc.
  • Per-character settings: All configuration is saved to INI files per server and character name.

Source File Structure

File Purpose
MQMyDPS.cpp Plugin entry point. Contains MyDPSEngine implementation: combat state management, damage and heal recording, spawn ID resolution, DoT cast tracking, MQMyChat integration, INI settings, and slash command handling.
MQMyDPS.h MyDPSEngine class declaration. Defines the central engine with battle tracking, active DoT registry, per-player heal targets, pet and non-melee counters, and spawn resolution methods. Declares IsMyChatLoaded() and SendToMyChat().
MyDPSData.h Core data structures: DamageType enum (includes DirectHeal), DamageRecord, TargetDamageData, HealTargetData, ActiveDoT, BattleData (with healTargets map), MyDPSSettings (with full color table including heal and target colors), and helper functions DamageTypeToString() and DamageTypeToColorKey().
MyDPSParser.cpp Chat line parser. 14 TryParse* methods using string_view::find() for fast pattern matching against all EQ combat and heal message formats. Includes TryParseDirectHeal which handles both the basic and spell-name formats.
MyDPSParser.h MyDPSParser class declaration with character and pet name configuration. Declares all 14 TryParse* methods.
MyDPSRenderer.cpp ImGui rendering. Combat spam overlay (newest-at-top, click-through support), main window with 5 tabs (Current Battle, History, Targets, Healing, Graphs), data-driven config window, font scaling on all windows, ImPlot charts. Color lookup uses the auto C = [&](key) -> ImVec4 lambda pattern throughout.
MyDPSRenderer.h MyDPSRenderer class declaration. Public render methods plus private per-tab helpers including RenderHealing().
MyDPSTLO.cpp TLO implementation. Exposes DPS data via ${MyDPS} top-level object with target lookup by spawn ID or name.
MyDPSTLO.h MyDPSType and MyDPSTargetType declarations with member/method enums.

Key Dependencies

Dependency What it provides
MQ2Main Plugin API, chat hooks, command registration, TLO framework, character/spawn data access, GetCombatState(), ParseMacroData() (used for MQMyChat integration), IsPluginLoaded().
ImGui Immediate-mode GUI for all windows, tables, tabs, menu bars, buttons, font scaling via PushFont/PopFont, and configuration panels. Menu bar uses MenuItem for compact layout.
ImPlot Plotting library built on ImGui. Used for DPS line graphs, damage bar charts, and pie charts (both damage and healing distribution). Axes are locked at minimum 0,0 to prevent negative-axis confusion.
fmt Modern C++ formatting library for string construction and number formatting. Also used to build MQMyChat message strings.
eqlib EverQuest game structures. Provides PlayerClient, LaunchSpellData (CastingData), EQ_Spell, ExtendedTargetSlot, and spawn type constants.
MQMyChat (optional) If loaded, receives battle summaries and reports via ParseMacroData("${MyChat.Send[MyDPS,message]}"). Not required — the plugin checks for it at settings load time and gracefully skips all chat sends if it is absent.

How to Build

  1. Place the MQMyDPS folder under plugins/ in your MacroQuest source tree.
  2. Add the MQMyDPS.vcxproj to the MacroQuest solution.
  3. Build with the Win32 (x86) platform configuration.
Note: MQMyDPS has no external vcpkg dependencies beyond what MacroQuest already provides (ImGui, ImPlot, fmt, eqlib). MQMyChat integration is optional at runtime — it requires no build-time dependency on that plugin.
